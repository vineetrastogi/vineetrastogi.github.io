How to use the "Group_by" Enumerable Method

November 19, 2014

This last week has involved a lot more intensive work dealing with Ruby and the seemingly endless number of methods that are available. Basically a method is a tool you can use which helps reach some desired output or store some required data.

If we wanted to look at an array and for example, find the number of elements in that array, you could use the method: .length.

array = [0,1,2,3,4,5]

array.length # returns 6, because there are 6 elements in the array

This is a basic example of using a method to return a characteristic of a data structure or object. What if we wanted to use more complex methods that could iterate over and manipulate multiple elements?

Enter, Enumerables.

One of the most useful Enumerables that I have come across this past week has been the Group_by method. Essentially what it does is returns a hash with values in arrays which are grouped according to certain parameters in the method's code block.

fruits = ["Apple", "Banana", "Plum", "Grape", "Apricot", "Peach", "Guava", "Boysenberry"]

first_letter = fruits.group_by { |x| x[0] }

# {"A" => ["Apple", "Apricot"], "B" => ["Banana", "Boysenberry"], "G" => ["Grape", "Guava"], "P" => ["Plum", "Peach"]}

As you can see, this specific example grouped an array of fruits by their first letters. It then created a hash to create keys and values for those letters and elements respectively. The code block { |x| x[0] } basically says that for each element x, I want you to pay attention to, and sort by the first letter (x[0]) in each element.

Overall, it's really cool to see that Ruby has so many built-in methods. It makes learning the language as a beginner so much more intuitive. I can ask myself, "What if I want to do this?" with any bit of code, and there is most likely a Ruby method that allows me to do that. I'm very excited to see how to utilize these methods later on.
