Arrays and Hashes

November 10, 2014

This week has been all about being able to go through the basics of what makes Ruby tick. I've had basic exposure to Java through classes in college, and can say that Ruby seems like a language that is much more intuitive and syntactically basic. (I might be eating these words in a few weeks though). Anyways, one of the more useful and interesting topics that I've learned about recently has been when to use an array vs. a hash.

Essentially the way I think about it is that both of these tools are useful, but can be utilized for different people based on how they think or what they need. Some people really like to have a related or ordered list of items which they can pull from at any point in time (arrays), and others like to have a list of items paired with similar items (hashes). This may be confusing at first, but hopefully when I'm done with this post we'll both have a better understanding.

Arrays are stored lists of values which are related to one another or are organized in a certain order.

Arrays can be created using: 

arr = Array.new

Or:

arr = [x, y, z]

This is an array with an ordered list of numbers 

 arr = [1. 2. 3. 4. 5. 6]

Here is an array with a list of different types of juices 

 arr = [apple, orange, grape, pineapple, guava]

You can access any value in an array at any time using this syntax: 

 arr = [apple, orange, grape, pineapple, guava]

arr[0] #=> apple

arr[3] #=> pineapple

There are a ton of methods in ruby that allow you to manipulate arrays. You can add and delete values, organize the values, combine arrays, etc. You get the point. There's a ton of stuff you can do with arrays, but the basic reason to use them is simple: Arrays are used to organize a set of values that make sense to be together.

Hashes are similar to arrays in the sense that they are used to store values, but are used for different purposes. Hashes are more associative in their usage. They are used to create pairs of values which are similar, rather than having a huge list of related values.

Heres a hash I'm creating that stores the grades of students in a class. 

grades = {:Elizabeth => 90, :Jimmy => 85, :Clayton => 76, :Samantha => 88}

You can also create hashes by using: 

grades = Hash.new

As you can see, hashes are used to assign values to certain elements. These pairs are called keys and values respectively. These values can be accessed similarly to arrays: 

grades[:Jimmy] #=> 85

grades[:Samantha] #=> 88

Hopefully you can now distinguish hashes from arrays, and their respective usages. These are definitely tools that are fairly reliant on preference by the programmer, and each have their own benefits and detriments.

Thanks for reading!
